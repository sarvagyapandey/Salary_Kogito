Database Representation

DATABASE
Master EMPLOYEE MASTER DATABASE

Employee Basic Details

  • Employee ID: String -- Unique ID for each employee; no duplicates.
  • Name: String -- Full name for slips, reports, and UI.
  • Date of Joining: Date -- Drives experience and benefits.
  • Date of Resignation: Date/null -- Null means still working.
  • Gender: Male/Female/Other -- Records and compliance.
  • Employment Type: FULL_TIME/CONTRACT/INTERN -- Impacts PF, gratuity, benefits.
  • Status: ACTIVE/INACTIVE -- Whether currently working.
  • Payroll Status: ACTIVE/HOLD/EXITED -- Controls payroll processing.

Academic Details

  • Highest Academic Qualification: String -- Highest degree obtained.
  • Branch: String -- Field of study (e.g., Computer Science, Commerce).
  • Year of Qualification: Integer -- Year completed.
  • Status of Qualification: COMPLETED/PENDING -- Completion state.
  • Professional Start Year: Integer -- Year professional work began.
  • Years of Experience: Integer -- Total experience from start year.

Certifications

  • Certification Name: String -- Name of certification.
  • Certification Date: Date -- Achievement date.
  • Certification Serial No: String -- Unique certificate number.

Skills

  • Skill Name: String -- Technology, tool, or domain skill.
  • Proficiency Level: BEGINNER/INTERMEDIATE/ADVANCED -- Self or manager rating.
  • Years of Experience: Integer -- Time spent using the skill.
  • Last Used: Date -- Most recent professional use.
  • Primary/Secondary: Boolean -- Whether core to the role.
  • Certification Ref: String/URL -- Optional proof or badge link.

Personal Details

  • Correspondence Address: String -- Current communication address.
  • Permanent Address: String -- Home address.
  • Personal Email ID: String -- Personal contact.
  • Official Email ID: String -- Company-provided email.
  • Phone Number: String -- Contact number.
  • Marital Status: Single/Married -- For HR records.
  • Passport No.: String -- Optional identifier.
  • Aadhaar No.: String -- Government ID.
  • PAN No.: String -- Required for income tax.
  • Father's Name: String -- Record keeping.
  • Mother's Name: String -- Record keeping.

Statutory & Bank

  • PF No.: String -- Provident Fund account number.
  • UAN No.: String -- Universal Account Number.
  • ESIC Card No: String -- Employee State Insurance ID.
  • Bank Name: String -- Salary credit bank.
  • Account No: String -- Bank account number.
  • IFSC Code: String -- Transfer identifier.
  • CTC: Integer -- Annual salary offered.
  • PF Option: 1–5 -- PF scheme selection.
  • Professional Tax: Integer -- State-based monthly tax.

Loan / Advance Details

  • Loan Name: String -- Type of loan (e.g., Personal Loan).
  • Loan Amount: Integer -- Total loan value.
  • No. of EMI: Integer -- Installment count.
  • Advance Salary: Integer -- Advance to be adjusted.

Job Details

  • Team Name: String -- Team the employee belongs to.
  • Designation: String -- Job role/title.
  • Base Location: String -- Work location affecting rules.
  • Department: String -- Functional department (IT, HR, etc.).
  • Grade Level: G1/G2/G3 -- Seniority level.
  • Cost Center: String -- Budget unit paying salary.
  • Business Unit: String -- Company division.

Project Details

  • Employee Id: String -- Unique identifier for the employee.
  • Employee Name: String -- Full name of the employee.
  • Project ID: String -- Unique identifier for the project.
  • Project Name: String -- Client or internal project name.
  • Duration: Date range -- Start and End date(null if ongoing).
  • Role/Title: String -- Position on the project.
  • Allocation %: Integer -- Efforts put in the project in terms of percentage.
  • Tech/Domain: String -- Technologies or business domain.
  • Notes: String -- Deliverables or scope summary.

Attendance

  • Salary To be Calculated: Y/N -- Whether salary should be processed.
  • Project ID: String -- Service order/project reference.
  • Official Days: Integer -- Total working days in month.
  • Worked Days: Integer -- Days actually worked.
  • Employee Present Days: Integer -- Days present.
  • LOP Days: Integer -- Loss of Pay days.
Payroll PAYROLL (MONTHLY SALARY)

Identification

  • Employee ID: String -- Links payroll to employee master.
  • Payroll Month: String -- Month of salary calculation.
  • Run ID: String -- Batch processing identifier.

Input Snapshot

  • CTC: Integer -- Copied to preserve history.
  • CCA: Integer -- Extra allowance used.
  • Category: String -- Category for rules.
  • Location: String -- Location for rule evaluation.
  • PF Option: 1–5 -- PF scheme used.
  • Professional Tax: Integer -- State deduction.
  • Employee PF Override: Integer/null -- Override instead of calculated PF.

Attendance

  • Salary To be Calculated: true/false -- Whether processed this run.
  • Official Days: Integer -- Total working days (26, 30, etc.).
  • Worked Days: Integer -- Days actually worked.
  • Present Days: Integer -- Days present.
  • LOP Days: Integer -- Loss of Pay days.

Salary Components

  • Basic: Integer -- Base salary from CTC.
  • Basic Stat: Integer -- Statutory basic for PF/ESI.
  • HRA: Integer -- House Rent Allowance.
  • Bonus: Integer -- Rule-based bonus.
  • Special Allowance: Integer -- Balancing component.

Employer Contributions

  • Employer PF: Integer -- Employer PF contribution.
  • Employer ESI: Integer -- Employer ESI contribution.
  • Gratuity: Integer -- Employer gratuity contribution.

Deductions

  • Employee PF: Integer -- PF deduction.
  • Employee ESI: Integer -- ESI deduction.
  • Medical Insurance: Integer -- Fixed deduction.
  • TDS: Integer -- Monthly income tax.

Tax Details

  • Tax Slab Base: Integer -- Base amount for tax.
  • Tax Multiplier: Decimal -- Applied tax rate.
  • Tax After Rebate: Integer -- Post-rebate tax.
  • Tax With Cess: Integer -- Final tax including cess.

Final Salary Values

  • Gross Payable (WITHOUT CCA): Integer -- Salary before deductions.
  • Annual Gross: Integer -- Yearly equivalent.
  • Take Home Salary (WITHOUT CCA): Integer -- Final amount after deductions.

System Fields

  • Payroll Status: PROCESSED/PENDING/FAILED -- Processing outcome.
  • Created At: Timestamp -- Record creation time.
  • Updated At: Timestamp -- Last update time.
